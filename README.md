### Commit convention

|태그 이름|설명|
|:---:|:---:|
|Feat|새로운 기능을 추가한 경우|
|Fix|버그를 고친 경우|
|Design| GUI를 수정한 경우|
|Refactor|프로덕션 코드 리팩터링|
|Comment|필요한 주석 추가 및 변경|
|Docs|문서를 수정한 경우|
|Rename|파일 혹은 폴더명을 수정하거나 옮기는 작업만인 경우|
|Remove|파일을 삭제하는 작업만 수행한 경우|

[태그] 내용의 형태로 커밋 메세지를 작성하며 80자를 넘어가지 않는다.

ex) [Feat] 발사대 모의기 발사 이벤트 처리 기능 추가
